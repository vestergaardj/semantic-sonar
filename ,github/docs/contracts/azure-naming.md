# Azure Resource Naming Convention

All resource names are generated by `infra/shared/naming.bicep`. Do not hardcode names anywhere.

## Pattern

```
{workload}-{resource-abbreviation}-{environment}-{region-short}
```

## Workload Name
Set once in `main.bicepparam` as `workload`. Example: `myapp`

## Environment Abbreviations
| Environment | Abbreviation |
|---|---|
| Development | `dev` |
| Staging | `stg` |
| Production | `prod` |

## Region Abbreviations
| Azure Region | Abbreviation |
|---|---|
| West Europe | `we` |
| North Europe | `ne` |
| East US | `eus` |
| West US 2 | `wus2` |

## Resource Type Abbreviations
| Resource | Abbreviation | Max Length | Example |
|---|---|---|---|
| Cosmos DB Account | `cosmos` | 44 | `myapp-cosmos-dev-we` |
| Function App | `func` | 60 | `myapp-func-dev-we` |
| App Service Plan | `asp` | 40 | `myapp-asp-dev-we` |
| Storage Account | `st` | 24 (no hyphens) | `myappstdevwe` |
| Key Vault | `kv` | 24 | `myapp-kv-dev-we` |
| Static Web App | `swa` | 40 | `myapp-swa-dev-we` |
| App Insights | `appi` | 260 | `myapp-appi-dev-we` |
| Log Analytics Workspace | `log` | 63 | `myapp-log-dev-we` |
| Resource Group | `rg` | 90 | `myapp-rg-dev-we` |
| Container Registry | `cr` | 50 (no hyphens) | `myappcrdevwe` |

## Notes
- Storage accounts and Container Registries cannot contain hyphens — `naming.bicep` handles this automatically
- Key Vault names must be globally unique — if collision occurs, append a short hash
- Never abbreviate the workload name — use it in full to keep names readable
